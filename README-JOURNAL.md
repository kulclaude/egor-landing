# Как добавить кейс в journal — без кода

Страница `journal.html` теперь берёт список кейсов и отзывов из одного файла — `cases.json`. Чтобы добавить кейс, не нужно трогать HTML или писать код — только отредактировать этот файл в GitHub и закоммитить.

## Про видео — где физически хранить ролик

**Не заливай сам .mp4-файл в репозиторий.** GitHub Pages — это статический хостинг для страниц, а не видеосервис: файлы будут грузиться медленно, есть лимит 100 МБ на файл, и нет нормального плеера из коробки.

Правильный путь — ролик уже и так лежит там, где ты его публикуешь:

- **Threads / Instagram** — просто возьми ссылку на сам пост с видео (кнопка «Поделиться» → «Скопировать ссылку»)
- **Telegram-канал** — ссылка на пост в канале тоже подходит

На лендинге кейс — это кликабельная карточка с обложкой и заголовком, которая по клику открывает оригинальный ролик там, где он опубликован (в Threads/Instagram/Telegram). Так видео грузится быстро, а сам ролик остаётся там, где ты его и так публикуешь.

Обложку (превью-картинку) можно не делать вообще — тогда карточка будет просто с заголовком на фоне. А можно один раз сохранить скриншот/обложку ролика (jpg или png, до ~300 КБ) и положить в папку `thumbs/` рядом с остальными файлами сайта.

## Как добавить кейс — шаг за шагом

1. Открой в GitHub файл `cases.json` в своём репозитории → нажми на иконку карандаша (Edit)
2. В массив `"cases"` добавь новый блок (не забудь запятую перед новым блоком, если он не первый):

```json
{
  "title": "Как я настроил Claude Code на Windows",
  "link": "https://www.threads.com/@egor.kulkov/post/XXXXXXX",
  "thumb": "thumbs/reel-01.jpg"
}
```

Если обложки нет — просто убери строку `"thumb"` целиком (или оставь пустую строку `""`).

3. Внизу страницы — Commit changes
4. Через 1-2 минуты GitHub Pages пересоберёт сайт, и кейс появится на странице `journal.html`

## Как добавить отзыв ученика

Точно так же, но в массив `"testimonials"`:

```json
{
  "name": "Диана",
  "text": "Прошла первые уроки, теперь настраиваю бота сама.",
  "link": "https://t.me/egorfromperm/123"
}
```

`"link"` необязателен — если ссылки нет, просто убери эту строку.

## Если захочешь совсем без ручных правок

Когда кейсов станет много и захочется автоматизировать полностью (например — чтобы Threads-агент сам добавлял сюда каждый новый пост) — это отдельная настройка через GitHub API и токен доступа к репозиторию. Скажи, когда будет актуально, сделаем.
